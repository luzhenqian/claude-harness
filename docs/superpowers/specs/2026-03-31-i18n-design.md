# i18n Design Spec

## Overview

Add internationalization to claude-harness supporting English (en), Simplified Chinese (zh), and Japanese (ja).

## Approach

Use `next-intl` with Next.js App Router.

## Changes

### Routing

- All pages move under `app/[locale]/`
- Default locale: `en` (with prefix)
- Middleware handles locale detection from `Accept-Language` header and redirects

### Translation Files

`packages/web/messages/{en,zh,ja}.json` — UI strings only (nav, titles, descriptions, labels).

### What Gets Translated

- Navigation links, page titles, descriptions, button labels
- Module descriptions in module-stats (via translation keys, not regenerating data)

### What Does NOT Get Translated

- Source code in code browser
- MDX articles (remain English-only for now)
- Generated JSON data (file-tree, module-stats structure)

### Language Switcher

Dropdown or button group in Nav component. Preserves current path when switching.

### File Structure Changes

```
packages/web/
├── src/
│   ├── app/
│   │   └── [locale]/          # all pages move here
│   │       ├── layout.tsx
│   │       ├── page.tsx
│   │       ├── articles/
│   │       ├── code/
│   │       └── modules/
│   ├── i18n/
│   │   ├── request.ts         # next-intl server config
│   │   └── routing.ts         # locale routing config
│   └── middleware.ts           # locale detection middleware
├── messages/
│   ├── en.json
│   ├── zh.json
│   └── ja.json
```
